Het analyseren van opslag in de cloud

macOS maakt het mogelijk om via Finder bestanden te bekijken die opgeslagen zijn in de cloud. Dit geldt voor bestanden die beheert worden met behulp van Apple's iCloud, maar ook voor cloud opslag oplossingen van derden zoals Dropbox, Google Drive en Microsoft's One Drive. Standaard zijn deze bestanden enkel opgeslagen in de cloud en wordt de inhoud opgehaald zodra dat nodig is.

Hoe dit werkt in combinatie met GrandPerspective wordt hieronder toegelicht.

Hoe beheert macOS de toegang tot "online only" cloud bestanden?

macOS definieert een "File Provider" extensie die applicaties kunnen gebruiken om toegang te geven tot bestanden die in de cloud zijn opgeslagen. Vanaf macOS 12.3 is dit de enige "on demand" oplossing voor het beheer van cloud data. Het wordt door alle populaire cloud opslag oplossing op macOS ondersteunt, waaronder iCloud, Google Drive, Dropbox, en One Drive.

Bestanden zijn standaard opgeslagen als "online only". Dit betekent dat ze geen ruimte innemen op de lokale schijf. Om de inhoud van zo'n bestand te bekijken wordt het bestand ge-download door macOS zodra dit nodig is. De inhoud kan vervolgens lokaal gecached worden zodat het een volgende keer snel beschikbaar is. Wanneer er te weinig schijfruimte is, kan macOS deze bestanden automatisch verwijderen om ruimte beschikbaar te maken.

Hoe bepaalt U hoeveel schijfruimte de cloud bestanden innemen?

Lees de "CloudStorage" map in met de fysieke bestandsgrootte maat. Omdat "online only" bestanden geen ruimte in nemen, zijn deze niet zichtbaar in het resulterende GrandPerspective overzicht. Het is daarom goed mogelijk dat dit overzicht enkel een kleine subset van alle bestanden laat zien.

Hoe bepaalt U hoeveel opslagruimte de cloud bestanden in de cloud innemen?

Lees de "CloudStorage" map in met de intrinsieke bestandsgrootte maat. Dit zal over het algemeen resulteren in een waarschuwing dat de weergegeven grootte groter is dan de werkelijke benodigde ruimte op de lokale schijf.

Kan het inlezen van een map door GrandPerspective resulteren in het downloaden van cloud bestanden?

Nee, GrandPerspective vraagt enkel bestandsmetadata (bestandsgrootte, tijd van laatste wijziging, enz) op van macOS. macOS hoeft de "online only" bestanden niet te downloaden om deze data te retourneren.

Echter, als de map nog niet eerder bekeken was in Finder of ingelezen door GrandPerspective, dan zal macOS de mappen structuur aanmaken tijdens het inlezen. Dit kan enige tijd in beslag nemen maar resulteert niet in het downloaden van cloud bestanden in de map.

Hoe kan ik ervoor zorgen dat GrandPerspective de mappen met cloud bestanden overslaat bij het inlezen?

U kunt hiertoe het filter "Geen cloud bestanden" gebruiken. Wanneer dit filter actief is worden de mappen met cloud bestanden tijdens het inlezen in hun geheel overgeslagen.

Dit filter kan het inlezen versnellen en is vooral handig als u veel cloud bestanden heeft en deze voornamelijk online staan. In dat geval kunt u zelfs overwegen het filter standaard aan te zetten. Dit kan u doen door het aanpassen van uw Voorkeuren.
